## Tech stack (locked decisions, do not re-litigate)

| Layer | Choice | Notes |
|---|---|---|
| HTML | Hand-written static in `src/static/` | No framework. Templates in `src/templates/` for blog pages. |
| JS bundler | **esbuild** | Only used to bundle `@meshsdk/core` for the browser. |
| Wallet SDK | `@meshsdk/core` 1.9.0-beta.103 | Bundled to `public/vendor/mesh.mjs` (~5.2 MB), lazy-loaded on first click. |
| Backend | **Cloudflare Worker** with `assets` binding + `run_worker_first: true` | One Worker, `src/worker.js`. Not Pages. |
| Chain data | **Koios** (`https://api.koios.rest/api/v1`) | Free, no API key. Used for pool stats + account info. |
| Chain data (fallback) | Blockfrost | Optional, only if `BLOCKFROST_KEY` Worker secret is set. |
| Content | WordPress WXR → markdown (frontmatter + body) → static HTML at build | Migration scripts in `scripts/`. |
| Fonts | Raleway (body) + Rubik (display) | Loaded from Google Fonts. |
| Hosting | Cloudflare Workers + Workers Assets | DNS on Cloudflare. Dev subdomain attached via dashboard. |

**Anti-stack:** the original brief said React + Vite + Pages. Ignore that. The house style is the esbuild + Worker + static-HTML pattern mirrored from `Z:\cardano\drep-learncardano`.

### Source-vs-generated rule
- **Edit `src/static/`, `src/content/`, `src/templates/`** — these are committed source of truth.
- **Never edit `public/`** by hand. `build.js` wipes it (except `.gitkeep`) at the start of every build. The generated files only exist on disk between builds and after deploy.

## Build pipeline

```
npm run build
  ├─ node build.js         → stamps src/version-data.js (git SHA + builtAt)
  │                          + clears public/, copies src/static/ → public/
  ├─ node build-mesh.js    → esbuild bundles @meshsdk/core → public/vendor/mesh.mjs
  └─ node build-posts.js   → reads src/content/posts/*.md (gray-matter + marked)
                             writes public/<slug>/index.html + /blog/ + categories
                             emits public/sitemap.xml
```

Order matters: `build.js` must wipe + copy first; the other two write into `public/`.

## Deploy

**SSH alias:** `aiagent-linux` (NOT `aiagent`). The host is `192.168.86.232`.
**Project path on aiagent:** `/home/aiagent/.openclaw/workspace/cardano/cardanode/` — this **is the same filesystem** as `Z:\cardano\cardanode\` on Windows (SMB mount). Files written from either side appear instantly on the other.

```bash

# = npm run build + npx wrangler deploy
```

### Auth gotchas
- Cloudflare auth is via `CLOUDFLARE_API_TOKEN` env var (not OAuth). It's exported in `~/.bashrc` on the aiagent box. **Non-interactive ssh** doesn't source `~/.bashrc` because it returns early for non-interactive shells (`case $- in *i*) ;; *) return;; esac`). Use `bash -ic '...'` to force an interactive shell.
- The token has Workers Scripts:Edit but **not** Workers Routes:Edit or DNS:Edit on the cardanode.com.au zone. Custom-domain attachment is done via the Cloudflare dashboard, not wrangler.
- `wrangler whoami` fails with the token (missing User:Read scope) but `wrangler deploy` works fine. Don't chase it.
- gh CLI is logged in as `pbwebdev` (HTTPS via stored token after `gh auth setup-git`). The aiagent SSH key is a deploy key — `git push` over SSH to new repos fails. Always use HTTPS remotes.

### Git on the shared mount
- **Run git on aiagent-linux**, not Windows. Windows refuses with "dubious ownership" because the SMB share is owned by the Linux uid. Don't waste time on `safe.directory` workarounds.

## Worker (`src/worker.js`)

| Route | Behaviour |
|---|---|
| `GET /` and `/index.html` | Fetches asset, runs HTMLRewriter to inject build SHA + live pool stats from Koios into `data-live="*"` placeholders. Also injects `noindex` on non-prod hosts. |
| `GET /api/pool-stats` | Koios `/pool_info` for ADAOZ. 24-hour edge cache. Returns `{ live_stake_ada, block_count, live_saturation, pledge_ada, margin, fixed_cost_ada }`. |
| `GET /api/account-info?stake=stake1...` | Koios `/account_info`. Validates stake address format, 60s cache. Returns `{ registered, delegated_pool, is_delegated_to_adaoz, total_balance_ada, rewards_available_ada }`. |
| `POST /api/contact` | Phase 6 placeholder — Turnstile verification then log. Email backend (MailChannels/Resend) not yet wired. |
| `GET /api/blockfrost/*` | Only active if `BLOCKFROST_KEY` secret is set. Optional fallback. |
| `GET /robots.txt` | Dynamic per host: prod = allow+sitemap; dev = `Disallow: /`. |
| Any other HTML | Fetches via assets binding, runs `maybeNoindex` to rewrite `<meta name="robots">` to `noindex, nofollow` on non-prod hosts. |
| Everything else | Pass-through to `env.ASSETS.fetch(request)`. |

**Production host detection** is by hostname (`cardanode.com.au`). All other hosts (`dev.cardanode.com.au`, `*.workers.dev`) get the noindex treatment.

### Cloudflare secrets

Set via `npx wrangler secret put <NAME>`:

| Secret | Purpose | Required? |
|---|---|---|
| `TURNSTILE_SECRET` | Verifies contact form submissions | For Phase 6 |
| `BLOCKFROST_KEY` | Optional fallback chain provider | No (Koios is primary) |

## Content workflow

### Adding a blog post
1. Drop a markdown file in `src/content/posts/<slug>.md` with frontmatter:
   ```yaml
   ---
   title: "..."
   slug: "..."        # must match filename
   date: "YYYY-MM-DD"
   categories: ["Cat 1", "Cat 2"]
   excerpt: "..."
   metaDescription: "..."
   ogTitle: "..."
   ogImage: "/uploads/..."   # optional; absolute URL or /-rooted
   youtube: ["https://..."]   # optional, sourced from body
   ---
   Body in markdown. YouTube iframes preserved verbatim. WP /uploads/ paths supported.
   ```
2. `npm run deploy` — build emits `public/<slug>/index.html`, updates `/blog/`, category pages, sitemap.
3. Comments file at `src/content/comments/<slug>.json` is optional (array of `{ id, parent, author, date, content }`).

### Re-migrating from WordPress
`scripts/migrate-posts.mjs` reads the largest `_temp/*.xml` automatically. Reruns are idempotent — same input = byte-identical output. Re-run with `npm run migrate` if the WXR export updates. **Markdown is committed**, so re-migration overwrites committed files.

### Pulling media
`scripts/pull-media.mjs` walks the markdown for `/uploads/...` references and downloads each from cardanode.com.au into `src/static/uploads/` (gitignored; ships via wrangler each deploy). 1 dead PDF logged at `_temp/media-missing.txt`.

## Frontend conventions

### Brand
- Primary blue `#0693e3` (decorative only). Text/button blue `--blue-strong: #0479b6` (4.79:1 on white = AA).
- Body `--ink: #0b1320`. Muted `--muted: #4a5568` (6.77:1 on white).
- Surface `#ffffff` / `--surface-2: #f4f7fb` (subtle) / `--surface-3: #e7f0f8` (operator section).
- Dark sections (Midnight + Delegate panel area can reuse this): `linear-gradient(180deg, #0b1320 0%, #16213a 100%)` with `#c8d3e6` body text (7.8:1 AAA).
- Fonts: Raleway 400/500/600/700, Rubik 500/600/700.

### Accessibility (WCAG 2 AA, audited)
- Global `:focus-visible` outline.
- `prefers-reduced-motion` kills transitions/transforms.
- Every interactive element ≥ 44px hit target. Buttons have `min-height: 44px`.
- Every decorative emoji has `aria-hidden="true"`.
- Every icon-only link has `aria-label` describing destination + "(opens in new tab)" where applicable.
- All external links open in new tabs with `rel="noopener noreferrer"`. CSS appends a `↗` glyph via `a[target="_blank"]::after` (suppressed inside `.btn`, `.link-card`, `.site-nav`).
- Blog post body external links: handled automatically by a `marked.use({ renderer })` link override in `build-posts.js`.

### SEO
- Canonical URLs always point to `cardanode.com.au` (production), even on dev. Dev host gets `noindex` so search engines never index the preview.
- Homepage carries a JSON-LD `@graph` with Organization, Person, LocalBusiness, Service, WebSite.
- Each post emits BlogPosting + BreadcrumbList JSON-LD.
- Each blog/category index emits CollectionPage JSON-LD.
- Geo meta: Brisbane lat/lon, `AU-QLD`.
- Open Graph + Twitter cards reference `/images/opengraph.jpg` (1200×630).
- `/sitemap.xml` regenerated at every build with home, /blog/, paginated pages, category pages + pagination, every post, and /contact-us/.

### Mesh With Us credits
- HTML comment ASCII-art block lives in `<head>` of every page (`src/static/index.html` + `src/templates/base.html`).
- Console banner + "type `mesh` anywhere" easter egg shipped in inline `<script>`.
- This is a global Peter convention — preserve it on every page.

## Delegation panel (Phase 4)

`src/static/scripts/delegate-panel.js` — vanilla JS state machine on the `#delegate` section. Lazy-imports `/vendor/mesh.mjs` on first click. States: `idle → picking → connecting → connected → working → success | already-adaoz | error`. Uses MeshSDK `BrowserWallet` for CIP-30 + `MeshTxBuilder` + `KoiosProvider` to build, sign, submit. First-time delegators get a stake-key registration cert (2 ADA deposit) bundled in the same tx as the delegation cert.

**Untested in production:** the Mesh 1.9 beta API may need method-name adjustments after first real wallet test. If anything fails on connect/build/submit, the browser console will show the exact missing symbol — easy patch.

## Common pitfalls

1. **Don't edit `public/*` directly** — wiped every build.
2. **Don't run git on Windows for this repo** — SMB ownership mismatch.
3. **Use `bash -ic 'cmd'` over ssh** so `~/.bashrc` is sourced and `CLOUDFLARE_API_TOKEN` is available.
4. **`wrangler whoami` returning 401 is fine** — the token lacks User:Read but has Workers:Edit. `wrangler deploy` works.
5. **Don't add the Cloudflare custom-domain via wrangler `routes`** — the token lacks zone perms. Use the dashboard.
6. **Reference repo is `drep-learncardano`**, not the original brief's React + Vite proposal.
7. **HTMLRewriter fires `<head>` element-open events BEFORE inner elements.** Don't try to detect "did I see a `<meta>`?" inside the head handler. Always replace, never conditionally-append.
