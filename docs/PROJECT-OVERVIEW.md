## Tech stack (locked, don't re-litigate)

| Layer | Choice | Notes |
|---|---|---|
| HTML | Hand-written static in `src/static/` | No framework. Page templates in `src/templates/` for blog pages. |
| JS bundler | **esbuild** | Used to bundle `@meshsdk/core` for the browser AND to minify CSS/JS at build time. |
| Wallet SDK | `@meshsdk/core` 1.9.0-beta.103 | Bundled to `public/vendor/mesh.mjs` (~5.2 MB), lazy-loaded on first click. |
| Backend | **Cloudflare Worker** with `assets` binding + `run_worker_first: true` | One Worker at `src/worker.js`. Not Pages. |
| Image processing | `sharp` (npm dev dep) | Used by `scripts/optimize-images.mjs` (WebP), `scripts/build-favicons.mjs` (favicon set + OG PNG re-encode). |
| Chain data | **Koios** (`https://api.koios.rest/api/v1`) | Free, no API key. Used for pool stats + account info + pool history + ticker lookup. |
| Chain data fallback | Blockfrost | Optional, only if `BLOCKFROST_KEY` Worker secret is set. |
| YouTube data | YouTube Data API v3 (env `YOUTUBE_API_KEY`) | Used by `/api/youtube`. **Must have "Application restrictions: None"** — CF Workers can't set Referer. |
| Email | Resend (env `RESEND_API_KEY`) + Telegram bot (env `TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID`) | Both optional. Contact form posts to both if configured. |
| Spam protection | Cloudflare Turnstile (env `TURNSTILE_SITEKEY` + `TURNSTILE_SECRET`) | Optional. If set, contact form requires token. |
| Content | WordPress WXR → markdown (frontmatter + body) → static HTML at build | Migration scripts in `scripts/`. 82 published posts. |
| Fonts | **Self-hosted** woff2 under `/fonts/` | Poppins 600/700 + Mulish 400/700 + JetBrains Mono. No third-party font requests at runtime. |
| Hosting | Cloudflare Workers + Workers Assets | DNS on Cloudflare. |

**Anti-stack:** the original brief said React + Vite + Pages. Ignore that. House style is the esbuild + Worker + static-HTML pattern mirrored from `Z:\cardano\drep-learncardano`.

### Source-vs-generated rule (re-stating because it bites people)
- **Edit `src/static/`, `src/content/`, `src/templates/`.** Source of truth.
- **Never edit `public/`** by hand — `build.js` wipes it (except `.gitkeep`) at the start of every build. Anything you save into `public/` disappears on the next `npm run build`.
- Image assets go in `src/static/images/`. WP-imported blog media goes in `src/static/uploads/` (`npm run media`).

## Build pipeline

```
npm run build
  ├─ node scripts/build-favicons.mjs   → renders favicon.svg → PNG set + ICO; re-encodes opengraph.png
  ├─ node build.js                      → stamps src/version-data.js (git SHA + builtAt)
  │                                       + clears public/, copies src/static/ → public/
  │                                       + esbuild-minifies public/styles/main.css and public/scripts/*.js in place
  ├─ node build-mesh.js                 → esbuild bundles @meshsdk/core → public/vendor/mesh.mjs
  ├─ node build-posts.js                → reads src/content/posts/*.md (gray-matter + marked)
  │                                       writes public/<slug>/index.html (82 posts)
  │                                       writes public/blog/index.html + pagination + category pages
  │                                       writes public/sitemap/index.html (HTML sitemap)
  │                                       writes public/sitemap.xml
  └─ node scripts/inline-css.mjs        → inlines minified main.css into <style> in every public/**/*.html
                                          and removes the <link rel="stylesheet"> tag entirely
                                          (kills the render-blocking CSS request PSI flagged)
```

Order matters — `build.js` must wipe + copy first; the others write into `public/`. `inline-css.mjs` runs last because it needs to patch every HTML produced earlier.

## Worker (`src/worker.js`)

| Route | Behaviour |
|---|---|
| `GET /` | Fetches asset, runs HTMLRewriter to inject build SHA, live pool stats (stake, delegators, blocks, ROA, saturation, sparkline SVG, pool ID), `?v=<sha>` on local CSS/JS URLs, noindex on dev hosts. |
| `GET /api/pool-stats` | ADAOZ stats via Koios `/pool_info` + `/pool_history`. 24h edge cache. Returns live + lifetime ROA + 20-epoch sparkline data. Cache key `v8`. |
| `GET /api/pool-info?id=...&ticker=...` | **Any** Cardano pool lookup. Ticker lookup via Koios `pool_list?ticker=eq.X` PostgREST filter. Used by the staking calculator's compare-pool feature. 6h cache. |
| `GET /api/account-info?stake=stake1...` | Koios `/account_info`. Validates input. Used by the delegation panel. 60s cache. |
| `GET /api/youtube` | YouTube Data API v3 → uploads playlist. **6h CF upstream cache; bump `_v=<n>` query param to bust if 403s stick.** Requires `YOUTUBE_API_KEY` with Application restrictions = None (Workers can't set Referer). |
| `POST /api/contact` | Honeypot + length validation + optional Turnstile verify. Posts to Telegram (if `TELEGRAM_BOT_TOKEN`+`TELEGRAM_CHAT_ID` set) AND/OR Resend (if `RESEND_API_KEY` set). `Promise.allSettled` — succeeds if any backend delivers. |
| `GET /api/blockfrost/*` | Only active if `BLOCKFROST_KEY` is set. Optional fallback. |
| `GET /robots.txt` | Dynamic per host: prod = allow+sitemap; dev = `Disallow: /`. |
| Any other HTML | `serveAsset` wraps `env.ASSETS.fetch` with path-aware Cache-Control + security headers + HTML transforms (`?v=<sha>` versioning, noindex on dev). |

**Production host detection** is by hostname (`cardanode.com.au`). All others (`dev.cardanode.com.au`, `*.workers.dev`) get the noindex treatment.

## Caching + versioning strategy

| Asset type | Cache-Control | Invalidation |
|---|---|---|
| HTML | `public, max-age=0, must-revalidate` | Always refetches; HTML rewriter stamps fresh `?v=<sha>` each request |
| `/styles/*.css`, `/scripts/*.js`, `/vendor/*.mjs` | `public, max-age=31536000, immutable` | Per-deploy `?v=<git-sha>` query string appended in HTML rewriter — browser sees new URL on each deploy, fetches fresh |
| Images, fonts | `public, max-age=31536000, immutable` | Filename change required. Wrangler Assets detects file content changes by hash and re-uploads; CF edge serves new bytes on next request. |
| `/sitemap.xml`, `/robots.txt`, other text | `public, max-age=3600` | TTL expires within an hour |
| `/api/pool-stats` | `public, max-age=86400` | 24h. Bump `cacheKey` version (currently `v9`) in worker to bust. |
| `/api/account-info` | `public, max-age=60` | 60s |
| `/api/pool-info` | `public, max-age=21600` | 6h |
| `/api/youtube` | `public, max-age=21600` | 6h. **Upstream CF cache is keyed by full URL — bump the `_v=<n>` query string in `worker.js` to force a fresh Koios/YT fetch.** |

### Security headers (sitewide via `withSecurityHeaders` in serveAsset)
- `x-content-type-options: nosniff`
- `referrer-policy: strict-origin-when-cross-origin`
- `permissions-policy: geolocation=(), microphone=(), camera=(), payment=(), usb=(), interest-cohort=()`
- `strict-transport-security: max-age=31536000; includeSubDomains; preload` (CF zone HSTS may override to `max-age=0` unless enabled in the dashboard)

## Content workflow

### Adding a blog post
1. Drop a markdown file in `src/content/posts/<slug>.md` with frontmatter:
   ```yaml
   ---
   title: "..."
   slug: "..."        # must match filename
   date: "YYYY-MM-DD"
   categories: ["Cat 1", "Cat 2"]
   excerpt: "..."
   metaDescription: "..."
   ogTitle: "..."
   ogImage: "/uploads/..."   # optional; absolute URL or /-rooted
   youtube: ["https://..."]
   ---
   Body in markdown. YouTube iframes preserved verbatim. HTML blocks pass through if there are no blank lines inside them.
   ```
2. `npm run deploy` — build emits `public/<slug>/index.html`, updates `/blog/`, category pages, both sitemaps.
3. Comments file at `src/content/comments/<slug>.json` optional.

### Embedding interactive widgets in posts
- HTML blocks in markdown pass through marked **only if there are no blank lines inside**. CommonMark spec: blank line ends an HTML block (type 6).
- The staking calculator (`/cardano-staking-calculator-rewards-ada/`) embeds raw HTML in the markdown body with no internal blank lines, then a `<script type="module" src="/scripts/staking-calculator.js"></script>` tag at the end. Script only loads on that one page because it lives in that post's body.

### Re-migrating from WordPress
- `scripts/migrate-posts.mjs` reads the largest `_temp/*.xml` automatically. Idempotent — same input = byte-identical output.

### Pulling media
- `scripts/pull-media.mjs` walks the markdown for `/uploads/...` references and downloads each from cardanode.com.au into `src/static/uploads/` (gitignored).

### Adding images
- Drop file in `src/static/images/<name>.<ext>`
- Run `npm run optimize` to generate `.webp` siblings (sharp, quality 75)
- Reference in HTML via `<picture><source srcset="/images/<name>.webp" type="image/webp"><img src="/images/<name>.jpg" ...></picture>` for static pages
- Blog post body images auto-wrapped in `<picture>` by marked renderer if a `.webp` sibling exists

### Updating favicons or OG image
- Edit `src/static/favicon.svg`
- Run `npm run build` — `scripts/build-favicons.mjs` regenerates the entire PNG/ICO set
- For OG image: drop `src/static/images/opengraph.png` (1200×630 recommended); the build step compresses any >500 KB PNG via sharp

## Common pitfalls (saves you from re-discovering them)

1. **Never edit `public/*` directly** — wiped every build. Source goes in `src/static/`.
2. **No git on Windows** — SMB ownership mismatch. Run git on aiagent-linux.
3. **Use `bash -ic 'cmd'` over ssh** so `~/.bashrc` is sourced and `CLOUDFLARE_API_TOKEN` is available.
4. **`wrangler whoami` returning 401 is fine** — token lacks User:Read scope; deploy works.
5. **Custom domains via dashboard, not wrangler** — token lacks zone perms.
6. **HTMLRewriter `<head>` element-open fires BEFORE inner elements.** Don't try to detect "did I see a `<meta>`?" inside the head handler. Always replace, never conditionally-append.
7. **Blank line inside an HTML block in markdown ends the block** (CommonMark). Keep embedded HTML chunks contiguous.
8. **Cloudflare Workers cannot set the `Referer` header** (forbidden by the Fetch spec). Google APIs with HTTP-referrer restrictions WILL fail from the Worker. Set Application restrictions to None on the API key; the key is in a Worker secret + API-restricted, so the security model is intact.
9. **CF upstream cache can stick** — if you change a Worker fetch and the response was previously bad, the bad response may be cached for the `cacheTtl`. Bump the `_v=<n>` cache-buster query string in the upstream URL to force a fresh fetch (see `/api/youtube`).
10. **The auditor parses every `<title>` element** — including SVG `<title>` tags inside icons/charts. Use `aria-label` on the parent rect/circle instead of inner `<title>` so SEO scanners don't read those as part of the page title.
11. **Image cache busting for immutable assets** — when you replace `peter-bui.webp` etc, wrangler detects the content-hash change and reuploads. Browser cache may still hold the old `1y immutable` copy until URL changes — usually a hard refresh fixes it. CF cache purge requires zone perms our token doesn't have.
